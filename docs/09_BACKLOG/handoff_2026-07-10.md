## 1. OBJETIVO / VISÃO

GastroMundi é um **SaaS white-label multi-estabelecimento de gestão para restaurantes/varejo**:
PDV + Caixa + Pedidos + Cozinha + Estoque + Financeiro + Clientes + Relatórios integrados, onde
a venda é a transação-fonte e uma IA transversal (Jarvas) observa tudo e sugere ações. Hoje
atende **um** estabelecimento (GastroMundi), mas a meta é **vender para vários** — cada cliente
com sua identidade visual, plano e dados isolados.

Problema de negócio: restaurantes operam com ferramentas desconectadas (PDV, planilha de
estoque, financeiro à parte). GastroMundi unifica tudo, e queremos **começar a vender na próxima
semana** — o que exige fechar isolamento multi-tenant, onboarding, console da plataforma e
fiscal (NFC-e).

- **Planos e gating por módulo (decisão 020/ADR-005)** `[DOC + EM CÓDIGO]`. 5 tiers superconjunto:
  Básico (Cardápio+PDV+Caixa) → Simples (+Estoque+Pedidos) → Médio (+Mesas/Comandas+Cozinha+

- **Billing e enforcement (decisão 022/ADR-006)** `[DOC + EM CÓDIGO]`. Decisões do dono:
  **bloqueio TOTAL** ao vencer, **carência de 3 dias**, status calculado NA CONSULTA (não em

  depender de gateway agora.
- **Add-ons NF-e e TEF são pagos e transversais (decisão 019)** `[DOC]`. Disponíveis em TODOS os
  planos por valor adicional (não são tier). Código nativo desde já; provedor pago adiado.
- **Theming/white-label (ADR-007)** `[DOC + EM CÓDIGO]`. Tokens `--gm-*` sobrescritos por

- **Driver de impressão trocável (decisão 025)** `[DOC + EM CÓDIGO]`. QZ Tray NÃO é dependência

- **Dois níveis de admin (decisão 027)** `[DOC — só decisão, sem código ainda]`. (a) Console da

- **Fiscal: provedor Focus NFe recomendado** `[SÓ CONVERSA — não confirmado como definitivo]`.

## 6. RESTRIÇÕES E CONTEXTO QUE NÃO PODEM SE PERDER

- **Ler `CLAUDE.md` e `memory/` antes de qualquer mudança.** Fonte de verdade. Se doc e código
  conflitarem, a doc prevalece (e deve ser corrigida).
- **Intuitividade é o Princípio nº 1** (inegociável). Estados sempre visíveis (carregando/vazio/
  erro/sucesso), prevenção de erro > mensagem de erro, alvos grandes no PDV (toque), rótulos em
  português do dia a dia, sem jargão. Justificar por que cada tela nova é intuitiva.
- **Stack real (ADR-004):** React 18 + Vite + Supabase direto (SDK, Auth, RLS) + React Router v6 +
  Context API (sem Redux) + Vercel. Schema em `supabase/schema.sql` + `supabase/migrations/`.
- **Segurança:** nunca hardcodar chaves/secrets — usar `import.meta.env.VITE_*`; secrets sensíveis
  (Anthropic, certificado fiscal) SÓ server-side / Edge Function; **RLS obrigatória** em toda
  tabela nova; **nunca** `select *` em tabelas sensíveis; validar inputs; nunca logar dado
  sensível.
- **Chave de papel no JWT (crítico — bug já corrigido):** usar SEMPRE
  `auth.jwt() -> 'app_metadata' ->> 'gastro_role'`. NUNCA `auth.jwt() ->> 'role'` (retorna
  'authenticated'). Guard: `rlsJwtClaimGuard.test.js`.
- **Papéis do app:** dono/gerente/caixa/atendente(garcom)/cozinha; +virá `plataforma`/super-admin
  (decisão 027). Papel vem de `app_metadata.gastro_role` via `custom_access_token_hook`.
- **Nomes de domínio em português** (ex.: `abrirCaixa`, `fecharComanda`, `baixarEstoque`,
  `registrarPagamentoFiado`); inglês só para padrões técnicos (`handleSubmit`, `useEffect`).
- **Padrões de código:** um componente por arquivo; erros de Supabase com try/catch ou `.error`;
  `activity_log` fire-and-forget (nunca bloqueia operação); funções puras (dinheiro, conversões,
  regras) nascem com teste; rodar `npm test` e `npm run build` antes de finalizar; CSS separado
  do JSX (`.css` co-localizado, tokens `--gm-*`/`alfa()`).
- **Venda é a transação-fonte (decisão 009):** finalizar venda dispara caixa/estoque/financeiro/
  vendas normalizadas/Jarvas via eventos. Qualquer mudança no fluxo de pagamento considera essa
  cascata.
- **Jarvas nunca executa ação sem confirmação humana**; não inventa números (fundamenta nos dados).
- **Custo:** priorizar gratuito; ao esbarrar em pago, apresentar custo+alternativa+recomendação e
  o dono decide (ex.: Focus NFe, certificado A1, QZ Tray com certificado, gateway).
- **White-label:** nada de marca/cor/logo/nome de um cliente hardcodado — vem do tenant.
- **Supabase do projeto:** `gastromundi-kora` (ref `jgdvylwiuqgjtavffguc`), free tier. Migrations
  pendentes rodam no SQL Editor; lembrar de habilitar RLS/Realtime no painel quando indicado.
