PDV que não trava quando cai a internet (Offline-First)

Sexta-feira, 10 de julho de 2026
Transformar o PDV numa PWA offline-first: as vendas continuam sendo lançadas normalmente mesmo sem Wi-Fi, gravando localmente e sincronizando com o Supabase automaticamente quando a conexão volta. Custo zero, impacto enorme no dia de pico.

1 Problema / Oportunidade

Em restaurante, a internet cai — roteador, operadora ou o provedor. Num PDV puramente online, isso significa não conseguir lançar pedido nem fechar comanda no meio do rush, ou seja, venda perdida e fila parada. Relatórios de mercado de 2026 apontam que operação offline confiável costuma ficar travada atrás de planos caros dos concorrentes. Entregar isso de graça é diferencial competitivo direto para o nosso SaaS white-label.

2 Como implementar (nossa stack)

  1. Virar PWA: adicionar vite-plugin-pwa (Workbox) — service worker com cache-first para assets e network-first para dados.
  2. Fonte de verdade local: usar Dexie.js (wrapper de IndexedDB) para guardar pedidos/itens localmente. A tela sempre lê do local → resposta instantânea.
  3. Fila de sincronização: toda escrita (novo pedido, item, fechamento) entra numa tabela outbox local com status pendente.
  4. Replay ao reconectar: ao voltar online, um worker envia a fila ao Supabase em ordem, marcando sincronizado. Usar upsert com ID gerado no cliente (uuid) para ser idempotente e evitar duplicidade.
  5. Feedback humano (Princípio nº1): badge visível no topo — 🟢 Online / 🟡 Offline (N pendentes) / 🔵 Sincronizando. Nada de erro assustador; o operador só continua vendendo.
// db.js — Dexie
import Dexie from 'dexie';
export const db = new Dexie('gm_pdv');
db.version(1).stores({
  pedidos: 'id, status, criadoEm',
  outbox:  '++seq, tabela, op, payloadId, status'
});

// registrar venda: grava local + enfileira (nunca bloqueia a UI)
export async function registrarPedido(pedido) {
  const id = crypto.randomUUID();
  await db.pedidos.put({ ...pedido, id, status: 'local', criadoEm: Date.now() });
  await db.outbox.add({ tabela: 'pedidos', op: 'upsert', payloadId: id, status: 'pendente' });
  return id;
}

// sincroniza ao voltar conexão
window.addEventListener('online', sincronizarOutbox);
async function sincronizarOutbox() {
  const pend = await db.outbox.where('status').equals('pendente').toArray();
  for (const item of pend) {
    const p = await db.pedidos.get(item.payloadId);
    const { error } = await supabase.from('pedidos').upsert(p); // idempotente pelo id
    if (!error) await db.outbox.update(item.seq, { status: 'sincronizado' });
  }
}

Nota: Background Sync API só existe em navegadores Chromium (Chrome/Edge). Para cobrir todos, dispare a sincronização no evento online + um setInterval leve como fallback. Lembrar de configurar RLS por tenant na tabela do Supabase.

3 Custo

100% gratuito. Dexie.js, vite-plugin-pwa e Workbox são open-source (MIT). IndexedDB é nativo do navegador. Nada muda no plano do Supabase — só reduz chamadas em rede. Sem alternativa paga necessária.

4 Benefício esperado

Zero venda perdida por queda de internet no horário de pico. PDV abre e responde instantaneamente (lê do local, ~95% mais rápido em recargas). Funciona em food truck, evento e loja com Wi-Fi ruim — casos reais do nosso público. Vira argumento de venda do SaaS.

5 Esforço & Intuitividade

Esforço: Médio (PWA fácil; a fila de sync exige cuidado com idempotência) Intuitividade p/ o operador: Alta — nada muda na tela, só um badge de status
