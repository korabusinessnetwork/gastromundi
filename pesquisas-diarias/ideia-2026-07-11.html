Resumo: Ler código de barras e QR direto no navegador com a BarcodeDetector API nativa — o operador aponta a câmera do próprio tablet/celular e o produto entra na comanda, sem leitor de mão, sem app e sem custo de licença.

1. Problema / oportunidade no PDV

No varejo e no balcão, digitar código ou procurar o item numa lista é lento e dá erro (item errado, preço errado). Leitores de código de barras USB/Bluetooth custam dinheiro, quebram e criam mais uma peça pra configurar por estabelecimento — contra o nosso alvo white-label de zero fricção. Como o PDV já roda no navegador (React+Vite/PWA), dá pra transformar a câmera que todo tablet/celular já tem em um leitor de código de barras grátis e instantâneo.

2. Como implementar na nossa stack

  1. Criar um componente <LeitorCodigo /> que abre a câmera com getUserMedia e roda a BarcodeDetector num loop com requestAnimationFrame.
  2. Ao detectar, buscar o produto no Supabase pelo campo codigo_barras (índice na coluna) e adicionar à comanda — com feedback humano (som + destaque + vibração).
  3. Fallback p/ iOS/Safari/Firefox (que não têm a API nativa): carregar sob demanda a lib grátis @zxing/browser. Assim funciona em qualquer aparelho.
  4. Debounce por código pra não somar o mesmo item 10×/segundo; prevenção de erro > correção.
// useLeitorCodigo.js — detecção nativa com fallback grátis
export function criarDetector() {
  if ('BarcodeDetector' in window) {
    return new window.BarcodeDetector({
      formats: ['ean_13', 'ean_8', 'code_128', 'qr_code', 'upc_a']
    });
  }
  return null; // -> cai no fallback @zxing/browser
}

async function lerFrame(detector, video, onLido) {
  const codigos = await detector.detect(video);
  if (codigos.length) onLido(codigos[0].rawValue); // ex: "7891000100103"

3. Custo

GRÁTIS A BarcodeDetector é nativa do navegador — zero licença, usa a aceleração de hardware do próprio SO. O fallback @zxing/browser é open-source (MIT), também grátis. Elimina a compra de leitores físicos (R$ 80–300 cada). Único "custo" é o tempo de dev.

4. Benefício esperado

  • Velocidade: item na comanda em ~1 s, sem digitar — fila anda mais rápido.
  • Menos erro: preço e produto vêm do cadastro, não da memória do operador.
  • Custo de setup por loja = 0: nada de hardware extra — combina com o white-label multi-tenant.
  • Serve também p/ QR: mesma tela lê QR de comanda de mesa, cupom ou Pix futuramente.
